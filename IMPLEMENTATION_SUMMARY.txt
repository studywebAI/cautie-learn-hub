KEY IMPLEMENTATION NOTES
================================================================================

DATA MODEL:
- Core FK chain: student_answers → blocks → assignments → paragraphs →
  chapters → subjects
- Handles mixed scoring: explicit score field OR is_correct boolean
- LEFT JOINs to handle incomplete hierarchies

QUERY PATTERNS:
- Use WITH (CTE) for complex aggregations (readability)
- Filter student_answers to last 90 days (performance)
- Add indexes on (student_id, submitted_at) if table is large
- Cache API responses for 1 hour (in-memory Map)

COMPONENT PATTERNS:
- All components are 'use client' (React client components)
- Use recharts for charts (already installed)
- Cards from UI library (@/components/ui/card)
- Icons from lucide-react (already installed)
- Responsive design with Tailwind classes

NOTES REMINDER LOGIC:
- localStorage key: `cautie:notes-reminder:dismissed:${topicId}`
- Value: timestamp (number), TTL: 30 days
- Shown only if: student has activity (quiz) but no notes on topic
- Dismissal can be cleared after 30 days to re-show reminder

ERROR HANDLING:
- Gracefully fallback if student_notes table doesn't exist
- Use safe navigation pattern: `.?.[0]?.property` for nested relations
- Implement try-catch in all endpoints
- Return empty structures on error (not 500)

PERFORMANCE:
- Implement caching at API level (1-hour TTL)
- Database indexes on foreign key columns
- Limit results to 90-day window
- Paginate teacher analytics (10 items at a time)

================================================================================
POTENTIAL PITFALLS & MITIGATIONS
================================================================================

1. Complex nested relations
   → Use safe navigation: const topic = answer.blocks?.[0]?.assignments...
   → Document the pattern clearly

2. N+1 queries
   → Use Supabase select() with all joins in one query
   → Test with explain/analyze if slow

3. Common mistakes extraction (open questions)
   → Only for block_type = 'open_question'
   → Aggregate by JSON value, truncate to top 3
   → Add error logging for debugging

4. localStorage key collisions
   → Use namespace: cautie:notes-reminder:dismissed:${topicId}
   → Document format clearly

5. Teacher access to other classes
   → Always filter by class_members.user_id = current_user
   → Verify in every teacher API endpoint

6. Dismissal flag not expiring
   → Check (Date.now() - storedTimestamp) < 30*24*60*60*1000
   → Clear if expired, show reminder again

7. Performance with large class
   → Limit teacher analytics to max 5 classes, 10 topics
   → Show warning if selection too broad

8. Student_notes table doesn't exist
   → Gracefully return hasNotes=false
   → Plan migration when ready
   → Don't block MVP on this
