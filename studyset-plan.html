Studyset — Redesign Plan

Making Studyset the center of the platform — connected to agenda, dashboard, tools, and calendars.

Core philosophy: A Studyset is not a folder of flashcards. It's a learning project — it has a goal, a schedule, linked tools, progress tracking, and a connection to class calendars and the dashboard. Every other tool (quiz, notes, flashcards, mindmap) flows through a Studyset.

Core Features

📄Materials library NEW

All uploaded files, links, and text pasted into any tool for this studyset are stored centrally and reusable. Open any tool and it pre-loads the studyset materials.

🔖Studyset detail page NEW

Rich single page: overview, progress ring, today's tasks, linked tool results (quizzes, decks, notes), schedule overview, and quick-launch buttons.

🎯Exam/deadline date NEW

Set an exam date. Triggers "Review" state automatically. Countdown shown on dashboard. Used to calculate study schedule intensity.

⚡Quick-launch tools NEW

From studyset detail: one-click to open Quiz / Flashcards / Notes pre-loaded with this studyset's materials. No re-pasting required.

📈Progress tracking NEW

Per-topic mastery ring (aggregated from quiz correct rate), flashcard review count, notes created. Combined into a studyset-level "readiness score."

🏷Subject + class linking IMPROVE

Link a studyset to a specific class (from the school timetable). This enables the class calendar ICS feed to pull deadlines automatically.

🔗Share studyset NEW

Generate a read-only share link. Other students can copy and make their own version. Optional: teacher-created shared studyset template for the class.

🗑Old 4-step creation wizard REMOVE

Replace with a lightweight modal: name + subject + optional exam date. Materials can be added after creation. Workflow types become a setting inside the studyset, not a creation gate.

Agenda Integration

Study Schedule → Agenda Blocks

When a studyset has a schedule (days + minutes/day), those blocks automatically appear in the Agenda as study events — distinct from class periods and assignments.

  • Color-coded by studyset (one color per studyset)
  • Click a study block → opens studyset quick-launch panel
  • Blocks auto-intensify in the week before exam date (Review state)

Assignment / Deadline Sync

If a studyset is linked to a class, deadlines from that class (set by teacher) auto-appear on the agenda. No manual entry needed.

Class Calendar — ICS Feed

The current calendar system is broken. The CalDAV connection flow (Apple/Google/Outlook) requires server credentials most users don't have. Almost nobody uses it. The correct approach for school calendars is a public subscription link — a URL that ends in .ics that any calendar app can subscribe to automatically.

How Apple / Google do it

Apple iCloud, Google Classroom, Outlook Teams all expose a stable, permanent URL per calendar:

# Apple iCloud example webcal://p01-caldav.icloud.com/published/2/... # Google Calendar example https://calendar.google.com/calendar/ical/class123%40group.calendar.google.com/public/basic.ics # Cautie proposed format https://app.cautie.nl/api/calendar/class/{classId}.ics https://app.cautie.nl/api/calendar/teacher/{teacherId}.ics # all classes combined https://app.cautie.nl/api/calendar/student/{token}.ics # student's own schedule

What the ICS feed contains

Event typeWho creates itShown to
Assignment deadlineTeacher (in Manage → Classes)All students in class
Quiz scheduledTeacherAll students in class
Exam / test dateTeacherAll students + parents
Class cancellationTeacherAll students
Studyset study blockAuto (from studyset schedule)Student only (private feed)

Security model

  • Class ICS (public-ish): Signed with a class token (opaque, long UUID). Teacher can rotate the token (invalidates old subscriptions). Link is not guessable but not secret — like Apple's approach.
  • Student ICS (private): Signed with a per-user token (stored in DB, not derivable). Contains personal study blocks. Separate URL from class feed.
  • Teacher combined feed: Merges all their class feeds into one. Useful for teacher's personal Google Calendar.
  • No login required to read ICS — calendar apps can't authenticate. Security is through unguessable tokens.
  • Rate limit: ICS endpoints are cached for 15 minutes. Calendar apps refresh every 30–60 min anyway.

How students subscribe

In Agenda settings: "Subscribe to class calendar." Shows the ICS link + a one-click button that opens webcal:// (which iOS, macOS, Google Calendar all understand natively). Student pastes the link once into their calendar app — done, auto-syncs forever.

Sharing

👁View-only share link NEW

Share a link to your studyset. Anyone with the link sees: materials list, notes, generated flashcards and quiz questions. Cannot edit.

📋Copy to my studysets NEW

From a shared studyset, one click to duplicate it into your own account with all materials, settings and tool results as a starting point.

🏫Teacher template NEW

Teacher creates a studyset template for a class. Students in the class see it in their studyset list as a "pinned" template they can activate.

🔒Privacy control NEW

Private (default) / Class only / Link shareable / Public. Materials are never exposed in public mode — only structure and tool outputs.

Progress & AI

Readiness Score

A 0–100% score per studyset, calculated from:

  • Quiz performance (40%) — correct rate on last 3 quiz sessions, weighted by recency
  • Flashcard mastery (30%) — % of cards at "known" level in spaced repetition
  • Coverage (20%) — how many topics in the material have been tested
  • Recency (10%) — how recently you studied (decays over 3 days)

AI suggestions

After a quiz or flashcard session, AI suggests: "You're weak on [topic]. Generate a focused quiz on just that section?" — links back to studyset with that topic pre-filtered. No extra AI cost beyond what the user already triggered.

Roadmap — Implementation Phases

Phase 1 — Foundation (do first, blocks everything else)

  • Simplify studyset creation: replace 4-step wizard with a quick modal (name + subject + optional exam date)
  • Add state, exam_date, class_id, last_activity_at to studysets table
  • Create studyset_materials table — central materials library per studyset
  • New studyset detail page — overview, materials, quick-launch tools, placeholder sections for progress
  • State machine: auto-transition Draft → Active when schedule set, In Progress when any tool runs

Phase 2 — Integration (connects studyset to the rest of the app)

  • Tag tool runs to studyset — studyset_tool_runs table
  • Dashboard card: "Today from [Studyset]" with real data, streak, countdown
  • Recents in tools show studyset breadcrumb
  • Agenda: study blocks from studyset schedule appear on agenda
  • Quick-launch: open any tool pre-loaded with studyset materials

Phase 3 — Calendar, Progress & Sharing

  • Class calendar ICS endpoint (/api/calendar/class/{classId}.ics)
  • Teacher UI to create calendar events (deadlines, exams, cancellations)
  • Student agenda "Subscribe" button with webcal:// link
  • Readiness score calculation + display on studyset detail page
  • Share link (view-only) + "Copy to my studysets"
  • Teacher template studyset for class

Decision Points — Need Your Input

1. Creation flow

Option A: Modal popup (name + subject + exam date) — fast, no wizard
Option B: Keep a simplified 2-step wizard (info → materials)
Recommendation: Option A. Users can add materials after creation, from inside the studyset.

2. Studyset list page layout

Option A: Cards grid (current, just improved) — good for visual overview
Option B: List with progress bars and "today's tasks" inline — more information density
Option C: Split: active studysets as large cards, archived as compact list
Recommendation: Option C.

3. Readiness score — show or hide?

The score only becomes meaningful after a few tool sessions. Do you want it visible from the start (starts at 0%) or only shown after first tool run?
Recommendation: Show after first quiz or flashcard session.

4. Class calendar: teacher approval needed?

Should the ICS feed for a class require teacher opt-in (teacher must press "publish calendar") or be automatically available for every class?
Recommendation: Auto-available but teacher sees it in class settings with a copy-link button.

5. Sharing privacy

When a student shares a studyset, should the original uploaded materials (PDF, text) be visible to the recipient, or only the AI-generated outputs (quiz questions, notes, flashcards)?
Recommendation: Only AI outputs by default. Materials are opt-in to share.
