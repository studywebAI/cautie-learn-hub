🔧 15 Universal Tool Layouts

Design Rules: Same layout structure for all tools (except Studyset later). Input at bottom/middle (never top). Options sidebar can be top/bottom/right (never left). Each layout is visually unique but structurally consistent.

LAYOUT 1: Notes Tool

NOTES - Layout 1: Vertical Split
Structure: Left sidebar (note list) + right editor
Input: Text editor at bottom right for new content
Options: Top right - font, formatting, AI tools
Why: Natural for sequential note-taking, common in Notion/OneNote
NOTES - Layout 2: Canvas View
Structure: Infinite canvas (like Miro/FigJam)
Input: Bottom input bar or click-to-add on canvas
Options: Right sidebar for block types
Why: Students with visual learning style prefer spatial layouts

LAYOUT 2: Flashcards Tool

FLASHCARDS - Layout 3: Card Stack
Structure: Large card centered, action buttons below
Input: N/A (review mode), separate creation panel
Options: Top right - deck selection, shuffle, settings
Why: Matches physical flashcard experience, minimizes distractions
FLASHCARDS - Layout 4: Grid View
Structure: Grid of cards, bulk editing mode
Input: Bottom add card form
Options: Right sidebar for filters/views
Why: Quizlet-style, good for managing large decks

LAYOUT 3: Quiz Tool

QUIZ - Layout 5: Question Focus
Structure: Question text centered, options below
Input: Answer selection buttons at bottom
Options: Top right - timer, question counter, difficulty
Why: Quiz-focused, minimal distractions
QUIZ - Layout 6: Detailed View
Structure: Question list left, detail/answers right
Input: Answer at bottom in main panel
Options: Top - filters, time spent, score
Why: Good for review mode, see all questions, track progress

LAYOUT 4: Mind Map Tool

MINDMAP - Layout 7: Radial
Structure: Central node, expandable branches
Input: Bottom form to add nodes/branches
Options: Right sidebar for colors, shapes, layout
Why: Natural for hierarchical thinking, visual learners
MINDMAP - Layout 8: Outline
Structure: Hierarchical outline, collapsible branches
Input: Bottom add-node form or inline editing
Options: Right sidebar for hierarchy levels, expand/collapse all
Why: Text-oriented students prefer this format

LAYOUT 5: Concept Map Tool

CONCEPTMAP - Layout 9: Node+Link
Structure: Nodes + relationship links on canvas
Input: Bottom form or click-to-add on canvas
Options: Right sidebar for relationship types, link labels
Why: Perfect for understanding relationships between concepts
CONCEPTMAP - Layout 10: Graph
Concept Links Type
Structure: Tabular view of nodes and connections
Input: Bottom add form or inline table editing
Options: Right sidebar for export, views, filters
Why: Data-heavy learners prefer tabular structure

LAYOUT 6: Timeline Tool

TIMELINE - Layout 11: Vertical
Structure: Central timeline, alternating left/right events
Input: Bottom form to add events
Options: Right sidebar for date, descriptions, categories
Why: History/chronological learning, visual narrative
TIMELINE - Layout 12: Horizontal
Structure: Horizontal timeline, stacked events above/below
Input: Bottom form or drag-to-add on timeline
Options: Right sidebar for zoom, date range, filters
Why: Better for dense timelines, scrollable

LAYOUT 7: Annotation/PDF Tool

ANNOTATE - Layout 13: Document Center
Structure: Left thumbnails/outline, right main document
Input: Highlight to annotate, right-click options
Options: Top - color picker, note types, search
Why: Same as traditional PDF readers (Adobe, Notability)
ANNOTATE - Layout 14: Full Screen
Structure: Document full-width, right sidebar for notes
Input: Highlight creates note in sidebar
Options: Top right - view mode, colors, search
Why: Document focused, minimizes switching between views

LAYOUT 8: Summary/Recap Tool

SUMMARY - Layout 15: Topic Breakdown
Structure: Sections with content blocks, expandable
Input: Bottom form to add sections/summaries
Options: Right sidebar for AI generation, tone, length
Why: Hierarchical organization, works for summaries/study guides

🚀 Setup Wizard Redesign

Current Issues: Glitches, duplicate screens, unclear text, poor security, not feeling complete. New Approach: Progressive onboarding, required steps only, save progress, optional later.

Wizard Flow (6 Required Steps)

1
School/Subject Selection
Choose what you're studying

Why This Step:

Personalizes the app to student's context. Used to recommend relevant tools and organize materials.

Questions (2-3 only):

  • What school/university are you at? (Autocomplete)
  • What subjects are you studying? (Multi-select, max 5)
  • What grade/year? (Dropdown)

⏱️ Time: 30 seconds | 💾 Auto-saved

2
Study Goals & Schedule
When and how much do you study

Why This Step:

Calculates smart spaced repetition schedules and helps with time management. Prevents overload.

Questions (3 only):

  • How much time do you study per day? (Radio: 15m / 30m / 1h / 2h+ )
  • When do you usually study? (Multi-select: Morning / Afternoon / Evening)
  • How many days until your first exam/test? (Date picker)

⏱️ Time: 45 seconds | 💾 Auto-saved

3
Create First Material
Start with what you have

Why This Step:

Gets students using the app immediately. Success early = higher engagement and retention.

Options:

  • Quick upload: Paste notes or upload PDF → AI creates flashcards
  • Create empty: Start with blank notebook/deck
  • Import: From Quizlet, Anki, or pasted text

⏱️ Time: 2-3 min | 💾 Auto-saved | 🤖 AI auto-processes in background

4
Create Study Schedule
When to review what

Why This Step:

Spaced repetition requires scheduling. Students don't know when to study what. We automate this.

Questions (2 only):

  • Should we auto-schedule your reviews? (Yes / Let me customize)
  • Reminder notifications? (Push / Email / Both / None)

⏱️ Time: 30 seconds | 💾 Auto-saved | 🔒 Optional (can set later)

5
Privacy & Security
Who can see your stuff

Why This Step:

Critical for student safety. Must be explicit. Vague privacy = liability and user distrust.

Clear Checkboxes:

  • ☑️ My materials are private (only I can see)
  • ☐ Share materials with my class (if applicable)
  • ☐ Allow teacher to see my progress
  • ☑️ Enable backup to cloud

⏱️ Time: 1 min | 💾 Must confirm before next | 🔐 Defaults to most private

6
You're Ready!
Start studying

On This Screen:

  • Summary of what was set up
  • Quick links to first material + first study session
  • "Skip for now" link to dashboard (can setup later)
  • FAQ section: "What is spaced repetition?" etc

⏱️ Total wizard time: ~5-7 minutes (can be completed in parts)

Key Improvements Over Current Wizard

❌ Old Problems:
  • Asked too many questions at once
  • Could repeat/glitch and show twice
  • Text was unclear/jargon-heavy
  • Unclear security implications
  • No way to skip and come back
  • Didn't get students using app immediately
✅ New Design:
  • Max 3 questions per step, 6 steps total
  • State saved after each step, no repeats
  • Simple language, no jargon
  • Explicit checkbox-based privacy
  • Can skip, resume on dashboard anytime
  • Step 3 = immediate action, AI auto-processes

⚙️ Tool Options/Settings (What to Include)

Based on Research: Only include features students actually asked for on Reddit/forums. Skip AI BS modes.

NOTES Tool Options

Option What It Does Why Students Want It
Rich Text Formatting Bold, italic, colors, lists, code blocks, tables Standard in OneNote/Notion. Makes notes scannable.
AI Summary Generates TL;DR of a note (not full BS AI) Reddit users asked for this - saves time reviewing notes
Auto-extract Questions AI turns note content into practice questions Bridges gap between notes → study. Huge time saver.
Markdown Support Write in Markdown, render as formatted text Tech students, CS majors prefer this
Shared Editing Collaborate with classmates in real-time Group studying, shared class notes

FLASHCARDS Tool Options

Option What It Does Why Students Want It
Spaced Repetition Auto-schedules based on difficulty + confidence Science-backed. 23% higher exam scores (studies show)
Multiple Formats Multiple choice, fill-blank, matching, image cards Not all studying is Q&A. Variety improves retention.
Image Support Add diagrams, charts, or images to cards Medical/science students need visual flashcards
Study Streaks Track consecutive days studied (visual counter) NOT gamification, just habit tracking. Works psychologically.
Cram Mode All cards once, no spaced repetition (before exam) Last-minute cramming happens anyway. Better to support it.
Audio Cards Add audio files or text-to-speech Language learning, commute studying, learning differences

QUIZ Tool Options

*(You've already done this manually, adding here for completeness)*

Option What It Does Why Students Want It
Timed Mode Question timer, section timer, total test timer Simulates real exam pressure. SAT/AP students need this.
Show Explanations After answering, show why answer is correct/wrong Learning from mistakes is critical. Not cheating.
Difficulty Filtering Only show easy/medium/hard questions Progressive learning. Builds confidence.
Shuffle Questions Randomize order and options Prevents memorization by position, improves understanding
Review Mode After quiz, can review answers with explanations Post-test learning = most effective time to learn

MINDMAP Tool Options

Option What It Does Why Students Want It
Multiple Layouts Radial, outline, organic, top-down Different subjects need different structures
Color Coding Assign colors to branches (theme-based) Visual learners need color differentiation
Collapse/Expand Fold branches to focus on one topic Reduces cognitive load, focuses studying
Export to Outline Convert mind map to text outline/document Study guides, essay prep. Bridge between visual and text.

CONCEPTMAP Tool Options

Option What It Does Why Students Want It
Relationship Labels Label links: "causes", "leads to", "related to", custom Specificity matters. "protein causes" ≠ "protein relates to"
Bidirectional Links Links go both ways (not directed) Some concepts interact bidirectionally (chemistry reactions)
Hierarchy Levels Define main vs secondary vs tertiary concepts Helps understand relative importance

TIMELINE Tool Options

Option What It Does Why Students Want It
Multiple Timescales Years, decades, centuries, or minutes/hours Different subjects use different scales
Categorization Color-code events by type/category Political events vs natural events vs culture (history)
Date Precision Exact date, month, year, or era (BC/AD) Some events only known by era or century

ANNOTATE Tool Options

Option What It Does Why Students Want It
Highlight Colors Yellow, pink, blue, green highlights + notes Color system helps organize information
Side Notes Add margin notes while reading Mimics paper annotation. Natural for reading.
Auto-generate Notes AI summarizes highlighted sections Reddit students asked for this - fast study guide creation
Full Text Search Search annotations across PDFs Large textbooks, finding earlier mentions
